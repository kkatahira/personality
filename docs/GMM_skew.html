混合正規分布モデル 歪みのある分布へのフィット

(2019/09/06 作成中)

ここでは，歪みのある分布へGMMをフィットすると 過剰に多めのコンポーネントが選択されるということを確認します。 また，その性質がサンプルサイズにどのように依存するか調べてみます。

準備

本ページで使用するパッケージがインストールされていない場合は，以下の コマンドをコンソールに入力してインスト―ルしてください。

install.packages("mclust")
install.packages("mvtnorm")
install.packages("sn") # skew-normal distribution distributionを扱うためのパッケージ
install.packages("tidyverse")

必要なパッケージの読み込み

以下で必要なパッケージを読み込みます。

library(mvtnorm)
library(mclust)
library(sn)
library(tidyverse) # ggplot用

データの生成

デモに使うデータを生成します。 ここではデータは2次元のデータであり，それぞれ独立に skew-normal distributionから生成します。

set.seed(1)
n.sample <- 1000

X1 <- rsn(n = n.sample, dp = c(-1, 1, 4))
X2 <- rsn(n = n.sample, dp = c(-1, 1, 4))

df_data <- data.frame(cbind(X1,X2))

データをプロットする

生成したデータを以下でプロットしてみます。

par(pty = "s")
plot(df_data$X1, df_data$X2, pch=".",
     cex = 2, xlab="X1", ylab="X2")

以下で推定した密度のカラーマップを作ります。

par(pty = "s")
# 最適なカーネルの幅 (bandwidth) を選択する
Hpi <- ks::Hpi.diag(x = df_data)

# カーネル密度推定
kd <- ks::kde(x = df_data, H = Hpi)

image(kd$estimate,
      x = kd$eval.points[[1]],
      y = kd$eval.points[[2]],
      xlab = "X1", ylab = "X2",
      col = viridis::viridis(20))

MclustによるGMMのフィッティング

BICにより，コンポーネント数を選択します。

BIC <- mclustBIC(df_data, 
                 G = 1:15, # 候補となるコンポーネント数
                 modelNames = "VVI"
                 )
plot(BIC)

コンポーネント数が3のときにBICの値が最大 (MClustの仕様では大きい方が良いモデル) となり，選択されます。

BICで選択されたモデルでモデルフィッティングをしてみます。

mod1 <- Mclust(df_data, x = BIC)

par(pty = "s")
plot(mod1, what = "classification")

データ生成に使った分布はクラスター構造はない単峰の分布なのですが，歪んだ構造 (裾野が大きい値の方に伸びている) を表現するために3つの正規分布のコンポーネントが使われています。一見，3つのクラスターに分かれているように見えますが，少なくとも上の二つのコンポーネントに実際に密度のピークがあるわけではないので解釈には注意が必要です。

サンプルサイズ依存性

上のようなGMMで選択されるコンポーネントがサンプルサイズに どのように依存するか調べてみます。

与えられたサンプル数のデータを生成して， BICでGMMのコンポーネント数を選択するという関数を作ります。

GMM_simulation <- function(n.sample) {
  
  X1 <- rsn(n = n.sample, dp = c(-1, 1, 4))
  X2 <- rsn(n = n.sample, dp = c(-1, 1, 4))
  
  BIC <- mclustBIC(data.frame(cbind(X1,X2)), G = 1:15, 
                   # 候補となるコンポーネント数
                   modelNames = "VVI",verbose = FALSE)
  which.max(BIC) # 最大のBIC (選択されたモデル) を返す
}

シミュレーションで試すサンプル数を以下のように指数関数で 作ります。(10を底とする対数軸で表示したときに等間隔になるように10のべき乗スケールで作ります)

n.sample.array <- round(10^seq(1.2,4.8,by=0.2))
print(n.sample.array)

各サンプルサイズでシミュレーショし，選択された

コンポーネント数を記録します。

df_simulation <- data.frame()

for (idxn in seq_along(n.sample.array)){
  cat("idxn:", idxn, "\n")
  
  # 各サンプルサイズにつき5回実行する
  for (idxr in 1:5){
    df_simulation <- df_simulation %>%
      rbind(data.frame(
        nsample = n.sample.array[idxn],
        K = GMM_simulation(n.sample.array[idxn]))
      )
  }
}
